# DevOps Interview Notes

## Purpose
Interview notes for CI/CD, infrastructure as code, release safety, and operational maturity.

---

## Core Concepts
- automated build and deployment pipelines
- infrastructure as code and environment consistency
- release approvals and quality gates
- blue-green or canary deployment strategies
- rollback readiness and monitoring

---

## Sample Architect Answer
I see DevOps as an architecture enabler because a good solution is not complete unless it can be built, deployed, monitored, and changed safely. I favor automation, repeatability, and release controls that reduce risk without making delivery overly bureaucratic.
