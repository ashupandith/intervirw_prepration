# Azure OpenAI Interview Notes

## Purpose
This file focuses on enterprise usage of Azure OpenAI, model access, security, and solution patterns.

---

## Key Topics
- model selection and deployment strategy
- embeddings and chat completions
- private data handling and enterprise controls
- rate limits, cost, and latency management
- grounding and responsible AI controls

---

## Sample Architect Answer
I position Azure OpenAI as a governed model-serving capability within a broader enterprise architecture. The model alone is not the solution; it needs orchestration, access control, grounding, evaluation, and monitoring around it.
