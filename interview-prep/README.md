# Interview Preparation Knowledge Base

## Purpose
This repository is your structured architect-level interview preparation system. It is designed for senior roles such as Azure Solution Architect, Senior Solution Architect, Cloud Architect, Enterprise Architect, and AI / GenAI Solution Architect.

The goal is not to memorize theory. The goal is to produce **crisp, enterprise-grade, interview-ready answers** with:
- architecture judgment
- business context
- trade-off thinking
- leadership maturity
- real Azure and .NET examples

---

## How to Use This Repository
1. Start with [PROFILE](PROFILE.md) and [strengths-and-positioning](strengths-and-positioning.md).
2. Revise the most relevant technical tracks under the `core/` and `ai-data/` folders.
3. Use `leadership/` and `interview-types/` for behavioral, stakeholder, and design rounds.
4. Before each interview, review:
   - top strengths
   - story bank
   - target company mapping
   - likely system design scenarios

## Answer Style Standard
For future interview Q&A, always structure answers like this:

- **Category:** main domain
- **File Name:** primary storage file
- **Subtopic:** exact concept or scenario
- **Answer:** crisp architect-level response
- **Store this in:** primary file path
- **Cross-reference:** secondary files if relevant

Example:

```markdown
Category: Azure Architecture
File Name: core/azure.md
Subtopic: Multi-region architecture
```

## Interview Quality Bar
Every answer should sound like:
- a person who has made platform decisions
- someone who understands delivery constraints
- someone who can lead teams and stakeholders
- someone who can balance speed, cost, reliability, and security

Avoid:
- textbook-only definitions
- junior developer wording
- overly long explanations
- tool-centric answers without business value
